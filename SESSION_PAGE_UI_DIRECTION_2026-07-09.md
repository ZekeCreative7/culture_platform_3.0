# 세션 페이지 UI 방향 제안

작성일: 2026-07-09

## 결론

세션 페이지에는 `워크보드 + 타임라인 + 상태 레일` 조합이 가장 적합하다.

단순 카드 그리드는 등록된 세션을 나열하기에는 좋지만, 처음 쓰는 사람에게 "이 페이지에서 무엇을 해야 하는가"를 충분히 알려주지 못한다. 반대로 너무 설명적인 온보딩 UI는 처음 한 번은 친절하지만, 운영자가 매일 쓰기에는 거슬린다.

추천 UI는 다음이다.

> 상단은 짧은 운영 흐름 레일, 본문은 상태별 세션 워크보드, 상세는 오른쪽 인스펙터 또는 드로어

## 가장 좋은 화면 구조

### 1. 상단: 운영 흐름 레일

역할:
- 처음 보는 사람에게 전체 흐름을 알려준다.
- 숙련자에게는 현재 병목을 보여준다.

추천 형태:
- 4단계 가로 레일
- 각 단계는 숫자와 상태만 보여준다.
- 긴 설명은 숨긴다.

단계:
1. 대상 구성
2. 일정 확정
3. 설문/응답
4. 리포트/후속 실행

좋은 UI 문법:
- 단계 번호는 작은 원형 배지
- 상태는 숫자 중심
- 위험 단계만 은은한 amber
- 완료 단계는 녹색보다 차분한 green tint

피해야 할 것:
- 설명 문장이 많은 카드 4개
- 큰 아이콘 4개
- 마케팅 랜딩 페이지처럼 보이는 히어로

이 화면은 운영 도구이므로 화려한 히어로보다 "오늘 어디가 막혔는지"가 더 중요하다.

### 2. 본문: 상태별 워크보드

현재는 유형별 그룹이다.

- 팀빌딩 6개
- 리더십 1개
- 협업 0개

이 구조는 관리자가 세션 유형을 이미 이해하고 있을 때는 괜찮다. 하지만 처음 쓰는 사람에게 더 중요한 기준은 유형이 아니라 상태다.

추천 기본 탭:
- 해야 할 일
- 진행 중
- 리포트 준비
- 전체 목록
- 캘린더

또는 보드형 컬럼:
- 일정 필요
- 설문 필요
- 응답 필요
- 리포트 가능
- 정상 진행

가장 직관적인 기본값은 `해야 할 일`이다. 사용자가 페이지에 들어오면 "지금 처리해야 하는 세션"부터 보여야 한다.

유형 필터는 보조 컨트롤로 둔다.

- 전체
- 팀빌딩
- 리더십
- 협업
- 커스텀
- 운영 서베이

## 카드 UI 제안

### 현재 카드의 문제

현재 카드에는 정보가 있다.

- 상태
- 유형
- 팀/기수
- 일정 확정
- 미확정
- 설문 응답 업로드
- 경고
- 다음 행동

하지만 시각적으로는 모두 비슷한 무게로 보일 수 있다. 사용자는 무엇이 핵심이고 무엇이 보조인지 한 번 더 읽어야 한다.

### 추천 카드 레이아웃

카드는 세 줄 구조가 좋다.

1. 제목 줄
   - 세션 이름
   - 운영 상태
   - 작은 수정/삭제 메뉴

2. 상태 트랙
   - 일정
   - 설문
   - 응답
   - 리포트

3. 다음 행동
   - 하나의 주 액션만 크게
   - 보조 정보는 작은 캡션

예:

```text
DT기획팀 2026
팀빌딩 · 운영 종료

일정 완료 | 설문 있음 | 응답 없음 | 리포트 불가

[응답 업로드]
사전/사후 응답을 넣으면 리포트가 준비됩니다
```

### 상태 표현 방식

가장 직관적인 것은 `칩 나열`보다 `미니 진행 트랙`이다.

추천:
- 각 단계는 작은 점 또는 짧은 pill
- 완료: solid muted green
- 진행 필요: amber outline
- 아직 불가: gray outline
- 실패/주의: red는 정말 위험할 때만 사용

예:

```text
대상 ●  일정 ●  설문 ●  응답 ○  리포트 ○
```

이 방식은 "왜 리포트가 안 되는지"를 글보다 빨리 보여준다.

## 세션 등록 UI 제안

### 가장 좋은 패턴: 선택 카드 + 단계형 드로어

현재 드로어는 강력하지만 한 번에 너무 많은 입력을 보여준다.

추천 구조:

1. 세션 유형 선택
2. 대상 구성
3. 일정 구성
4. 설문 준비

### 세션 유형 선택 화면

드롭다운보다 카드형 선택이 낫다.

이유:
- 유형이 단순 값이 아니라 운영 목적이 다르다.
- 초심자는 `리더십`, `협업`, `커스텀`의 차이를 이름만 보고 알 수 없다.

추천 카드:

- 팀빌딩
  - 한 팀의 분위기, 신뢰, 협업을 다룰 때
  - 팀장 + 팀원

- 리더십
  - 여러 팀 리더가 같은 이슈를 맞출 때
  - 팀장 그룹

- 협업
  - 리더십 이후 실행 그룹을 만들 때
  - 여러 팀 구성원

- 커스텀
  - 특정 대상만 별도로 구성할 때
  - 직접 구성

- 운영 서베이
  - 문화 세션 밖 단발 점검
  - 안전/시설/만족도

카드에는 `추천 사용 상황`을 한 줄로 넣는다. 기능 설명보다 선택 판단을 돕는 문장이 중요하다.

### 드로어보다 인스펙터가 더 좋아 보이는 경우

세션을 새로 만들 때는 드로어가 좋다.

하지만 기존 세션을 볼 때는 오른쪽 인스펙터가 더 고급스럽다.

추천:
- 카드 클릭: 오른쪽 상세 패널 열림
- 수정 버튼: 편집 모드 전환
- 다음 행동 버튼: 해당 화면으로 이동

이렇게 하면 목록 화면이 "관리 테이블"이 아니라 "운영 콘솔"처럼 보인다.

## 캘린더 UI 제안

캘린더는 보조 탭으로 유지하는 게 맞다. 다만 현재 캘린더는 일정 확인용이고, 세션 운영 보드와 연결감이 약할 수 있다.

추천:
- 기본은 보드
- 캘린더는 "이번 주 일정" 중심
- 일정 클릭 시 출석 체크 또는 세션 상세 인스펙터

월간 캘린더보다 운영자는 보통 다음이 더 유용하다.

- 오늘
- 이번 주
- 다음 주
- 미확정 일정

따라서 캘린더 탭의 첫 화면도 월간보다 주간/리스트 혼합이 더 직관적이다.

## 색과 시각 품질

현재 플랫폼은 파란색 계열이 강하다. 세션 페이지는 파란색만 계속 쓰면 기능 구분이 약해진다.

추천 색 역할:
- Blue: 기본 정보, 이동, 활성 상태
- Green: 준비 완료
- Amber: 처리 필요
- Red: 삭제/위험만
- Slate/gray: 아직 조건 미충족

중요:
- 경고를 모두 amber로 칠하면 화면 전체가 불안해 보인다.
- red는 삭제, 무결성 오류, 데이터 손상 위험에만 쓴다.
- 세션 유형 색은 너무 강하게 쓰지 말고 좌측 얇은 라인이나 작은 태그로만 사용한다.

## 가장 추천하는 최종 UI

### 데스크톱

1. 상단 페이지 헤드
   - 세션 운영 보드
   - 짧은 설명
   - 새 세션 버튼

2. 운영 흐름 레일
   - 대상 구성
   - 일정 확정
   - 설문/응답
   - 리포트/후속 실행

3. 상태 필터 바
   - 해야 할 일
   - 진행 중
   - 리포트 준비
   - 전체
   - 캘린더

4. 세션 워크보드
   - 카드 기본 정렬은 `처리 필요도` 순
   - 카드에는 상태 트랙과 다음 행동 하나

5. 오른쪽 상세 인스펙터
   - 세션 클릭 시 상세
   - 편집은 명시적으로 전환

### 모바일

1. 운영 상태 패널은 정상일 때 한 줄
2. 페이지 제목과 새 세션 버튼을 먼저 보여준다.
3. 운영 흐름 레일은 2x2 미니 카드 또는 가로 스크롤
4. 기본 탭은 `해야 할 일`
5. 카드에는 다음 행동 버튼만 크게
6. 상세는 바텀시트

## 지금 코드에서 다음 UI 슬라이스로 추천하는 것

### 1순위: 카드 내부를 상태 트랙형으로 바꾸기

현재 다음 행동 버튼은 좋아졌지만, 카드의 상태 정보는 여전히 텍스트 중심이다.

다음 작업:
- `일정 확정 4/6`
- `설문 응답 업로드 0/2`
- `설문 데이터 없음`

이걸 아래처럼 바꾼다.

```text
대상 완료 | 일정 필요 | 설문 있음 | 응답 필요 | 리포트 대기
```

또는 미니 점 트랙:

```text
대상 ● 일정 ○ 설문 ● 응답 ○ 리포트 ○
```

### 2순위: 목록 기본값을 유형 그룹에서 해야 할 일 그룹으로 바꾸기

기본 진입:
- 일정 필요
- 설문 필요
- 응답 필요
- 리포트 가능

보조 보기:
- 유형별
- 캘린더

### 3순위: 새 세션 등록 첫 화면을 카드형 선택으로 바꾸기

드롭다운을 없애기보다, 드로어 상단에 유형 선택 카드를 먼저 둔다.

초심자는 카드로 고르고, 숙련자는 이후 편집에서 드롭다운을 써도 된다.

## 한 문장 원칙

세션 페이지 UI는 `설명하는 화면`이 아니라 `다음 행동이 보이는 운영 보드`여야 한다.

예쁘게 보이는 핵심은 장식이 아니라, 상태 위계가 분명하고 클릭할 이유가 자연스럽게 보이는 것이다.
