# Culture Platform 3.0 — 작업 회고 키트

> 목적: 여러 채팅 창(여러 세션)에 걸쳐 만든 이 플랫폼의 개발 과정을 돌아보고,
> **무엇을 잘했고 / 무엇을 잘못했고 / 다음엔 어떤 프로세스·어떤 에이전트로 할지**를 정리한다.
>
> 작성 시작일: 2026-06-25

---

## 왜 이 폴더가 있나

작업이 여러 채팅 창에 흩어져 있어서, 한 사람 또는 한 AI가 모든 창을 한 번에 볼 수 없다.
그래서 **각 창에서 같은 형식으로 정보를 뽑아낸 뒤 → 마지막에 하나로 통합**하는 방식으로 회고한다.

이 폴더는 그 작업을 위한 문서 세트다.

| 파일 | 역할 | 언제 쓰나 |
|---|---|---|
| `README.md` | 사용법(이 문서) | 시작할 때 한 번 |
| `SESSION_REVIEW_TEMPLATE.md` | 창 1개당 1부씩 채우는 템플릿 + 붙여넣기용 프롬프트 | 각 창에 들어갈 때마다 |
| `sessions/README.md` | 세션 기록 파일명·저장 규칙 | 세션 기록을 저장하기 전 |
| `FINAL_INTEGRATED_REVIEW.md` | 모든 창 결과를 합치는 최종 리뷰 | 모든 창 정리 끝난 뒤 |
| `KNOWN_WORK_SUMMARY.md` | 현재 저장소/기억 기준으로 이미 확인된 작업 흐름 요약 | 최종 리뷰의 초기 재료 |
| `NEXT_AGENT_WORKFLOW.md` | 다음 개발 때 쓸 권장 프로세스와 에이전트 역할 | 다음 프로젝트/다음 큰 기능 시작 전 |

채운 세션 기록은 `feedback/sessions/` 안에 번호를 붙여 저장한다.

## 3단계 워크플로우

### 1단계 — 각 창에서 정보 뽑기

1. 회고할 채팅 창을 연다.
2. `SESSION_REVIEW_TEMPLATE.md` 맨 위의 **"이 창에 붙여넣을 프롬프트"**를 복사해 그 창에 붙여넣는다.
3. 그 창의 AI가 자기 대화를 같은 형식으로 요약하게 한다.
4. 나온 결과를 `feedback/sessions/NN-주제.md`로 저장한다.
5. 가능하면 마지막 커밋/푸시 여부, 검증 여부, 되돌림 여부를 꼭 남긴다.

핵심은 모든 창에서 **같은 템플릿 구조**로 뽑는 것이다. 그래야 나중에 합쳐진다.

### 2단계 — 세션 기록 모으기

- `feedback/sessions/` 안에 창 개수만큼 파일을 쌓는다.
- 시간 순서대로 번호를 매긴다. 순서가 곧 개발 타임라인이 된다.
- 정확한 순서가 기억나지 않으면 파일 안에 “순서 불확실”이라고 적는다.

### 3단계 — 최종 통합 리뷰

- 모든 세션 기록이 모이면 `FINAL_INTEGRATED_REVIEW.md`를 연다.
- 세션 기록에서 사실과 해석을 끌어와 빈칸을 채운다.
- 마지막에 **다음 프로젝트 권장 워크플로우 / 에이전트 전략**을 결론으로 적는다.

이 마지막 통합 단계는 모든 세션 기록을 한 창에 넣고,
“이 기록들을 `FINAL_INTEGRATED_REVIEW.md` 형식대로 통합 분석해줘”라고 시키면 AI가 도와줄 수 있다.

파일명은 완벽할 필요 없다. 중요한 것은 번호와 주제가 보이는 것이다.

## 채울 때 지킬 원칙

- **사실과 해석을 구분한다.** “무엇을 했다”와 “그게 좋았다/나빴다”를 섞지 않는다.
- **재작업·되돌림은 반드시 기록한다.** 가장 비싼 교훈이 거기 있다.
- **칭찬도 기록한다.** 잘된 패턴을 다음에 재사용하는 게 회고의 절반이다.
- **검증 흔적을 남긴다.** “확인했다”가 아니라 어떤 화면/데이터/브라우저/PDF/모바일 폭에서 확인했는지 적는다.
- **점수는 솔직하게.** 좋게 보이려고 부풀리면 다음 결정이 틀어진다.
- 한국어로 작성한다.

## 최종적으로 답해야 할 질문

이 회고의 목적은 단순한 작업 일지가 아니라, 다음 개발 때 더 좋은 시스템을 만드는 것이다. 최종 리뷰에서는 특히 아래 질문에 답한다.

- 나는 무엇을 빠르게 잘 결정했나?
- 내가 중간에 자주 놓친 것은 무엇인가?
- AI가 헷갈린 지점은 어떤 정보가 부족해서였나?
- 어떤 단계에서 “기획 에이전트 / 구현 에이전트 / 검증 에이전트 / 릴리즈 에이전트”를 분리했으면 좋았나?
- 다음 프로젝트에서는 어떤 순서로 작업하면 재작업이 줄어드나?
- 어떤 완료 기준을 미리 정해야 “끝났다”고 말할 수 있나?
