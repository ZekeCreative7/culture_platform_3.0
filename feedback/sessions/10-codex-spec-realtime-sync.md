## 3. 산출물 (사실)
- 핵심 기술 결정:
  - **소프트 삭제 패턴 확립**: `deleteSurvey()` → `setSurveyDistributionActiveInFirestore()` (응답 데이터 보존)
  - **`rowMatchesSurvey()`** 공유 매칭 함수로 고아 스캔 + 응답 표시 로직 통일
  - **`updateResponsesSubscription`**: sessionIds에 삭제된 세션의 surveyId까지 포함하도록 확장
  - **실시간 동기화**: 주요 컬렉션에 `onSnapshot` 적용 → 멀티 디바이스 동시 사용 지원
- GitHub Pages 배포 이력:
  - `.nojekyll` 추가 전: Jekyll 빌더가 실패하며 이후 모든 배포가 큐에서 블록됨
  - 추가 후: `pages-build-deployment` 액션 자동 통과, 라이브 사이트 정상 서빙
